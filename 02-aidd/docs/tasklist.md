## Итерация 1 — Бот в Telegram

Цель: в чате с ботом видно, что он живой и реагирует на сообщения.

- [x] Запуск процесса бота с валидным конфигом
- [x] Ответ пользователю на явное действие в чате (например команда или первое сообщение)

**Как проверить:** написать боту в Telegram — получить ожидаемый ответ.

---

## Итерация 2 — Ответ через LLM

Цель: на текстовый вопрос приходит ответ, сформированный моделью.

- [x] Связка «сообщение пользователя → запрос к модели → ответ в чат»
- [x] Настройки модели задаются через конфигурацию приложения

**Как проверить:** задать вопрос боту — ответ по смыслу от модели, не заглушка.

---

## Итерация 3 — Роль ассистента

Цель: ответы соответствуют заданной роли из системного промпта.

- [ ] Роль задаётся в настройках и участвует в каждом запросе к модели
- [ ] Смена роли в конфиге заметно меняет стиль или содержание ответов

**Как проверить:** два разных описания роли — два разных «характера» ответов на один и тот же вопрос.

---

## Итерация 4 — Ошибки и пустой ввод

Цель: бот не падает; пользователь получает понятную реакцию при сбоях и мусорном вводе.

- [ ] Сбой обращения к модели: безопасное сообщение пользователю, детали только в логах
- [ ] Пустой или заведомо неподходящий ввод: корректная реакция без обрыва диалога

**Как проверить:** симулировать недоступность API и отправить пустое/бессмысленное сообщение — бот отвечает стабильно.

---

## Итерация 5 — Память диалога

Цель: ответы учитывают недавние реплики в рамках общения с пользователем.

- [ ] В запрос к модели попадает ограниченная история последних сообщений
- [ ] Лимит истории задан разумно (не раздувает контекст бесконечно)

**Как проверить:** ссылка на предыдущую фразу в следующем сообщении — модель «помнит» контекст.

---

## Итерация 6 — Запуск и проверки «из коробки»

Цель: новый человек может поднять бота и прогнать базовые проверки без разбора внутренностей.

- [ ] Описано, какие переменные окружения нужны для первого запуска
- [ ] Из корня проекта доступны команды установки, запуска, линтера и тестов
- [ ] Опционально: тот же сценарий через контейнер, если это входит в объём MVP

**Как проверить:** чистое окружение → по инструкции запуск → тесты и линтер проходят.
